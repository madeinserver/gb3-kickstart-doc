Overview of ManualOcclusionService

The ManualOcclusionService is a simple service that allows you to show or hide sets of entities under program control.

Each entity that will be shown or hidden by the service must have an integer property OcclusionRegion that sets its region. The service tracks all entities and builds lists for each region. When a region is shown or hidden via the builtin functions, the service makes the corresponding entities visible or hidden.

Typically you will need to set "trigger regions" in your level. When the player enters a trigger region, a list of occlusion regions will be shown or hidden.

ManualOcclusion Lua Bindings

SparkManualOcclusion.HideRegion(region)
Makes all entities marked for a particular region non-visible. Each entity that contains the property 'OcclusionRegion' with a value set to the value of the argument will have their 'Visible' property set to false. The intended use is for a trigger to be set between regions that will call this built-in to optimize the number of objects being drawn. 'true' is returned if any entities were found.
SparkManualOcclusion.ShowRegion(region)
Makes all entities marked for a particular region visible. Each entity that contains the property 'OcclusionRegion' with a value set to the value of the argument will have their 'Visible' property set to true. The intended use is for a trigger to be set between regions that will call this built-in to optimize the number of objects being drawn. The 'region' parameter is simply a number that matches the number set in the OcclusionRegion property of a set of entities. 'true' is returned if any entities were found.
