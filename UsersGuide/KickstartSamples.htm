Kickstart Samples

Kickstart ships with two sample applications, AlphaGame and DayDreamer. This page describes each sample and lists the controls for the sample.

Playing Alpha Game

Alpha Game could be considered a "GamePack". It consists of an executable, art, code, and other data that together create a very simple 3rd-person walkaround scenario. Alpha Game is built from a game solution that currently includes AlphaGame.emtproj, SparkLibrary.emtproj, and a level file called AlphaGame.block. The solution also references the Gamebryo Standard Library, the PhysX Standard Library for trigger collision, and the Wwise Solution for 3D sound.

You run Alpha Game by double-clicking its executable: <AlphaGame>\Application\win32\AlphaGame_Release_<VC80 or VC90>.exe

where:

  • <AlphaGame> is the directory that contains your local copy of Alpha Game
  • <VC80 or VC90> is shorthand for the version of Visual Studio that was used to compile the executable

The gameplay in AlphaGame is very simple. Using a standard Xbox controller on PC, you can:

  • Zoom the attached camera system in/out and around with the right analog stick (arrow keys on keyboard).
  • Move the player with the left analog stick (a,w,s,d keys on keyboard). Note that the direction in which the character moves is relative to the position of the camera.
  • The A button (spacebar on keyboard) is the "action" key. Currently, this is used to open the treasure chest.
  • The Start button (escape key on the keyboard) pauses and unpauses the game. Once paused, the only action currently available is to unpause.
  • Enemies patrol way points until they are within a certain distance of the player. Then they pursue the player and explode if they touch him, doing damage.
  • The game is won by reaching the end of the level and pressing the action button to open the treasure chest.
  • The game is lost if the player's health reaches zero.
  • There are also trigger and door examples placed thoughout the level.
  • Left shoulder button (LB) cycles through a series of performance tracker screens
  • Left trigger button (LT) dumps accumulated performance information to the log, and resets the counters. The first time you press it it will include performance information gathered during startup. Subsequent presses include information since the last press.

Playing DayDreamer

DayDreamer could be considered a "GamePack". It consists of an executable, art, code, and other data that together create a simple over-the-shoulder perpective shooter. DayDreamer is built from a game solution that currently includes DayDreamer.emtproj, SparkLibrary.emtproj, and two level files. The solution also references the Gamebryo Standard Library, the PhysX Standard Library for trigger collision, and the Wwise Solution for 3D sound. The Scaleform GFx library is also available although it is currently not being used.

You run DayDreamer by double-clicking its executable: <DayDreamer>\Application\win32\DayDreamer_Release_<VC80 or VC90>.exe

where:

  • <DayDreamer> is the directory that contains your local copy of DayDreamer
  • <VC80 or VC90> is shorthand for the version of Visual Studio that was used to compile the executable

The controls for DayDreamer are as follows:

  • Move the player with the left analog stick (a,w,s,d keys on keyboard).
  • Move the camera perspective and direction the player faces with the right analog stick, mouse, or direction keys on keyboard
  • Use the right mouse button to zoom
  • Fire with the left mouse button or right trigger button
  • Enable/Disable PhysX debug mode with F6
  • Change weapons by the mouse scroll or the left and right shoulder buttons
  • Press F3 to enter free-fly mode
  • Press Space or the gamepad A button to jump
  • Press "C" or the gamepad B button to crouch
  • Press "U" or the gamepad Y button to activate items
  • Press "Esc" or the gamepad Start button to bring up the menu.

Note that you must have ammunition to fire weapons other than the pistol. Ammunition pickups are scattered throughout the level.
