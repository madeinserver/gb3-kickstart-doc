Creating a Block File

Most games comprise more than one level. When you need to create a new level, you add a new .block file to your Game Solution.

Creating a new level

  1. If it is not already running, start Toolbench. From the Windows Start menu, select Emergent > Gamebryo LightSpeed > Tools > Toolbench.
  2. From the Toolbench menu, select File > Open Content and navigate to the directory that contains the Alpha Game solution file (for example, <AlphaGame>\Application\Data\AlphaGame.gsl).
  3. In the Game Solution View, right-click the AlphaGame.gsl file and select Add > New Content.
    This opens the New Content dialog.
  4. In the New Content dialog, click the "Block Document" item in the Content Types listbox.
  5. In the Name field, type a name for the .block file and then click OK. (We're using "My New Level" in this example.)
    A new block file will be created in the game solution with the name that you specified. World Builder may display some prompts about creating required directories. You can accept the defaults. You may experience a slight pause as World Builder switches over to the new block file.

Preparing the new level for Alpha Game

  1. In the next step of this tutorial, you'll be viewing the new level within Alpha Game. So, let's go ahead and drag some entities from the Palette View into the level. For best results, the following are the minimum number of entities to include:
    • FloorSection
    • DirectionalLight
    • SparkThirdPersonRotatingCamera
    • SoundScene
    • WallSection
    • PlayerCharacter

      Figure 2

  2. From the Toolbench menu, select File > Save All.

Obviously, this is a simplification. But you get the idea: Create entity models, add instances of those models in levels, separate game levels into unique .block files.
