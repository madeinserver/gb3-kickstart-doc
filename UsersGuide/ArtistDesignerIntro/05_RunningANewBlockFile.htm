Running a New Block File

In the previous step in this tutorial, we created a new level and added some entities to it. We can now point the Alpha Game executable at this level. To do this, we'll modify the Alpha Game configuration file.

Note: For a .block file to work with Alpha Game, it must contain a SparkCamera and a PlayerCharacter. That means that the SparkCamera model must be mixed into an entity in the .block file, and the PlayerCharacter model must be either instantiated as an entity or mixed into an entity in the .block file.

Running Alpha Game with your new level

  1. On your workstation, we recommend that you make a copy of the Alpha Game configuration file:
    <AlphaGame>\Application\win32\Config.ini
  2. Save this copy, perhaps using an obvious name like "Config_ORIGINAL.ini".
  3. Open Config.ini in a text editor and locate the following section:
    [Game]
    InitialWorld = urn:emergent-world:AlphaGame.
  4. Change the InitialWorld parameter as follows:
    [Game]
    InitialWorld = urn:emergent-world:<YourLevelName>.
    For example:
    [Game]
    InitialWorld = urn:emergent-world:My New Level.
  5. Save the file.
  6. Run Alpha Game by double-clicking its executable: <AlphaGame>\Application\win32\AlphaGame_Release_<VC80 or VC90>.exe

Obviously, this is a simplification. But you get the idea: Create entity models, add instances of those models in levels, separate game levels into unique .block files.
