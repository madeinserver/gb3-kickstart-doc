Adding Entity Models

You have the option to override model properties when you create entity instances in your world. But, if you want something that can serve as a game object template (for example, if you plan to re-use a game object in many levels and/or games), you will want to create an entity model. In this section, we'll investigate how to add a model (or preset) to Alpha Game.

Note: Gamebryo LightSpeed uses the term model and entity model to define a template of constructs that may or may not include a physical 3D model. The term does not refer to only 3D vertex data. It refers to any properties or game play constructs created in EMT that define the overall entity and its behavior.

Adding a new entity model

  1. If it is not already open, go ahead and open the Alpha Game solution file: <AlphaGame>\Application\Data\AlphaGame.gsl.
    When you double-click this file in Windows Explorer, it opens within Toolbench.
  2. In the Game Solution View, expand the tree below the Solution node and then double-click the Alpha Game EMT project (AlphaGame.emtproj).
  3. Now go ahead and double-click the "EntityModels" node to see all of the presets and models that are contained within the project.
  4. Double-click the preset called "FloorSection02-4Squares." (The blue diamond beside the entity means that it is a preset; a green diamond means that it is a model.) In the entity relationship diagram that displays, you can see that this preset mixes in the "Mesh" model, which is part of the from the Standard Model Library. You can also see that the Mesh model's asset is set to a NIF file.

    Figure 1

  5. Now that you've had a quick look at a model, let's go ahead and create a new one. In the Game Solution View, right-click the AlphaGame.emtproj node and select Add > New Model.
    The model is created in the Game Solution View and it appears within the entity diagram.
  6. If it is not already selected, click your new model in the diagram.
  7. In the Game Solution View, scroll down to the Standard Model Library EMT project (StandardModelLibrary.emtproj) and expand the tree so that you can see the models.
  8. Find the Mesh model in the list of Standard Model Library models, left-click on it, and drag it into the window above your new model. You have now mixed in the Mesh model to your new model.
  9. In the Entity Model Editor View, expand the Mixin Properties section and then, under the "Mesh" model heading, click the Browse button beside the NifAsset property.
    You can use "cube.nif", within <AlphaGame>\Application\Data\Graphics\. Or, if you have other NIF files that you want to use, copy them anywhere under the <AlphaGame>\Application\Data\ folder.

    Figure 2

  10. From the Toolbench menu, select File > Save All.

You have just created a new entity model that you can now place in World Builder. If you want to import multiple NIF files or actors, you can use the Model Factory to automatically create and name multiple presets for you.
