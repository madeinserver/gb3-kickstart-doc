Description

The PostEffectsStep contains a set of PostEffect instances that it renders in sequence.

Effects are added to the step by their constructors.

After creating an PostEffectsStep using the PostEffectsManager, insert it into your render frame after the main render step you want to apply post effects to, then set their output targets so that your main render step now outputs to your PostEffectsStep::GetInputTarget(), and your post effect step now outputs to the main render step's original target.

Manually sets the internal ping-pong buffer for post effects.

These are the buffers automatically selected for Early and Final passes. By default these are PostEffectsManager::k_bufferDefault0/1. Note that iBuffer0 may share a rendered texture as the input buffer, but iBuffer1 may not.
